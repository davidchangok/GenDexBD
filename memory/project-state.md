## 核心设计规则（来自代码逻辑）

1. **配招枚举**：按槽位分组（前3=主技能，后3=副技能）→ 枚举2³=8种配招 → 每Build独立标签 → 每Breed取最优Build分（消除幽灵配招）
2. **COMMUNITY > FORCE**：Score() 检测 COMMUNITY_BREED_BONUS 存在时跳过 FORCE 标签
3. **单品种自动设最佳**：JournalUI label() 中 numPossibleBreeds==1 自动 SetBestBreed
4. **否定词过滤**：NEGATE_PATTERNS 防止"阻止回复生命"误匹配 SCALES_HEALTH
5. **B/B共识映射**：单字母'B'→'B/B'（而非之前的缺失bug）
6. **歧义品种扣1分**：Breed 10(P/B)与8(P/S)系数完全相同时，BREED_AMBIGUITY声明8优先，10扣1分
