无论哪种操作，能用 skills 就用 skills，不要手工替代。

**Why:** 手工做出来的质量低（如手工 code review 质量远不如 skill），skills 会使用更结构化的流程、多角度并行检查、更严格的验证。
**How to apply:** 遇到 review → `/code-review` skill，遇到 debug → `systematic-debugging` skill，遇到设计 → `brainstorming` skill，遇到计划 → `writing-plans` skill。不要自己手工替代。
